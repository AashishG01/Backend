Secrets of JAvaScirpt

🔑 1. Everything is an Object (Almost)

Arrays, functions, and objects are all objects. Even primitives can act like objects temporarily (boxed).

🔄 2. Execution Context & Call Stack

JavaScript is single-threaded and uses an execution context model. The call stack tracks where the interpreter is in the code.

🧠 3. Lexical Scope

Scope is determined by where variables/functions are written in code, not where they’re called.

🚦 4. Hoisting

var and function declarations are moved to the top of their scope. let and const are hoisted but remain in the Temporal Dead Zone (TDZ) until defined.

🎯 5. Closures

A closure is when a function “remembers” its outer lexical scope even after the outer function has finished executing.

🧩 6. First-Class Functions

  • Functions can be assigned to variables
  • Passed as arguments
  • Returned from other functions

⛓️ 7. this Keyword

this refers to how a function is called, not where it's defined. In arrow functions, this is lexically inherited.

🌀 8. Asynchronous JavaScript

  • Event loop handles async code using:
  • Callbacks
  • Promises
  • Async/Await
  • Non-blocking I/O and task queues are key

🧱 9. Prototype Chain

JavaScript uses prototypal inheritance. Objects inherit from other objects via __proto__ or Object.prototype.

🔐 10. Modules & Encapsulation

Modern JavaScript (ES6+) supports import/export. Modules help with scope, reuse, and encapsulation of code.
