# FastAPI - Complete Backend Guide

> **Core idea:** FastAPI is a modern, high-performance Python web framework for building APIs. It leverages Python type hints to provide automatic validation, serialization, and interactive documentation. It is built on top of Starlette (for the web layer) and Pydantic (for data validation), making it one of the fastest Python frameworks available - on par with Node.js and Go.

## Why FastAPI?

```
The problem with traditional Python frameworks:

  Flask (2010):
    -> No built-in validation (you write it manually or use Flask-Marshmallow)
    -> No automatic API docs
    -> No async support (WSGI-based, synchronous)
    -> No type checking at runtime
    -> You end up installing 10+ extensions for basic features

  Django REST Framework (2011):
    -> Heavy, opinionated, lots of boilerplate
    -> Steep learning curve
    -> Slower performance (synchronous WSGI)
    -> Great for full-stack apps, overkill for pure APIs

  FastAPI (2018):
    -> Built-in validation via Pydantic (automatic, from type hints)
    -> Auto-generated interactive docs (Swagger UI + ReDoc)
    -> Async support native (ASGI-based, like Node.js event loop)
    -> Type hints = editor autocomplete + runtime validation
    -> Performance comparable to Node.js and Go
    -> Minimal boilerplate, maximum productivity
```

### The Key Insight - Type Hints Do Everything

```python
# In Flask, you write validation MANUALLY:
@app.route('/users', methods=['POST'])
def create_user():
    data = request.get_json()
    if not data.get('name'):
        return {'error': 'name required'}, 400
    if not data.get('email'):
        return {'error': 'email required'}, 400
    if not isinstance(data.get('age'), int):
        return {'error': 'age must be integer'}, 400
    # ... 20 more lines of validation
    return {'user': data}, 201


# In FastAPI, type hints DO the validation:
@app.post('/users', status_code=201)
def create_user(user: UserCreate):   # <-- Pydantic validates AUTOMATICALLY
    return {'user': user}

# If name is missing -> automatic 422 error with details
# If age is a string -> automatic 422 error with details
# If email is invalid -> automatic 422 error with details
# ZERO manual validation code needed!
```

## Core Concepts - How FastAPI Works

### The Stack

```
Your FastAPI Code
       |
       v
  Pydantic (data validation + serialization)
       |
       v
  Starlette (web framework layer - routing, middleware, requests)
       |
       v
  Uvicorn (ASGI server - like Nginx for Python async apps)
       |
       v
  uvloop (high-performance event loop, based on libuv)
       |
       v
  Operating System (TCP sockets, networking)
```

### ASGI vs WSGI

```
WSGI (Web Server Gateway Interface) - Flask, Django:
  -> Synchronous: one request at a time per worker
  -> While waiting for DB query -> worker is BLOCKED
  -> To handle 100 concurrent requests -> need 100 workers
  -> Each worker = separate process = lots of RAM

ASGI (Asynchronous Server Gateway Interface) - FastAPI:
  -> Asynchronous: many requests concurrently per worker
  -> While waiting for DB query -> worker handles OTHER requests
  -> To handle 100 concurrent requests -> 1 worker is enough
  -> Like Node.js event loop - non-blocking I/O

  This is why FastAPI can handle 7-10x more requests
  with the SAME hardware as Flask/Django.
```

### Pydantic - The Validation Engine

```
Pydantic is the SECRET WEAPON of FastAPI.

What it does:
  -> Takes Python type hints
  -> Validates incoming data against those types AT RUNTIME
  -> Converts data to the correct types automatically
  -> Generates JSON Schema (used for auto docs)
  -> Serializes response data

Without Pydantic (manual validation):
  if not isinstance(data['age'], int):
      raise ValueError('age must be int')
  if data['age'] < 0:
      raise ValueError('age must be positive')
  if not data.get('email'):
      raise ValueError('email required')
  # ... endless validation code

With Pydantic (declarative validation):
  class User(BaseModel):
      name: str
      email: EmailStr
      age: int = Field(gt=0)

  # Done. Pydantic validates EVERYTHING automatically.
  # Invalid data -> instant 422 error with detailed messages.
```

### Route Groups with APIRouter

```python
# In production, you DON'T put every route in main.py.
# You organize routes into separate files using APIRouter.

# routes/users.py
from fastapi import APIRouter

router = APIRouter(
    prefix="/api/v1/users",     # all routes start with this
    tags=["Users"],              # groups them in docs

from routes.products import router as product_router

app.include_router(product_router)

# Now:
#   GET /api/v1/users       -> get_all_users()
#   GET /api/v1/users/42    -> get_user(42)
#   POST /api/v1/users      -> create_user()
```

## Path Parameters

```python
# Path parameters are parts of the URL that change per request.
# FastAPI extracts them and VALIDATES their type automatically.

@app.get("/users/{user_id}")
def get_user(user_id: int):    # <-- type hint = int
    return {"user_id": user_id}

# GET /users/42    -> { "user_id": 42 }       (auto-converted to int)
# GET /users/abc   -> 422 Error: "value is not a valid integer"
#                     FastAPI rejected it because "abc" is not an int!


# Multiple path parameters:
@app.get("/users/{user_id}/posts/{post_id}")
def get_user_post(user_id: int, post_id: int):
    return {"user_id": user_id, "post_id": post_id}

# GET /users/5/posts/12  -> { "user_id": 5, "post_id": 12 }


# Enum path parameters (restrict to specific values):
from enum import Enum

class UserRole(str, Enum):
    admin = "admin"
    user = "user"
    moderator = "moderator"

@app.get("/users/role/{role}")
def get_users_by_role(role: UserRole):
    return {"role": role, "message": f"Fetching all {role.value}s"}

# GET /users/role/admin     -> works
# GET /users/role/hacker    -> 422 Error (not in enum)
```

## Query Parameters

```python
# Query parameters are key-value pairs AFTER the ? in the URL.
# Any function parameter NOT in the path is treated as a query param.

@app.get("/products")
def get_products(
    category: str = None,        # optional (has default None)
    min_price: float = 0,        # optional (default 0)
    max_price: float = 10000,    # optional (default 10000)
    sort_by: str = "created_at", # optional (default "created_at")
    page: int = 1,               # optional (default 1)
    limit: int = 20,             # optional (default 20)
):
    return {
        "filters": {
            "category": category,
            "price_range": [min_price, max_price],
        },
        "sort_by": sort_by,
        "page": page,
        "limit": limit,
    }

# GET /products?category=electronics&min_price=100&page=2&limit=10
# -> All params auto-parsed and type-validated!
# -> If someone sends page=abc -> 422 error automatically


# Required query params (no default value = required):
@app.get("/search")
def search(q: str):   # <-- NO default = REQUIRED
    return {"query": q}

# GET /search          -> 422 Error: "field required"
# GET /search?q=hello  -> { "query": "hello" }


# Query parameter validation with Query():
from fastapi import Query

@app.get("/items")
def get_items(
    q: str = Query(min_length=3, max_length=50),          # length limits
    page: int = Query(default=1, ge=1),                    # >= 1
    limit: int = Query(default=20, ge=1, le=100),          # 1-100
    tags: list[str] = Query(default=[]),                    # list of strings
):
    return {"q": q, "page": page, "limit": limit, "tags": tags}

# GET /items?q=ab           -> 422 (min_length is 3)
# GET /items?page=-1        -> 422 (must be >= 1)
# GET /items?limit=500      -> 422 (must be <= 100)
# GET /items?tags=python&tags=fastapi -> tags = ["python", "fastapi"]
```

## Request Body and Pydantic Models

### Defining Models

```python
from pydantic import BaseModel, Field, EmailStr
from typing import Optional
from datetime import datetime

# Pydantic model = the SHAPE of your data
class UserCreate(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    email: EmailStr
    age: int = Field(gt=0, lt=150)
    bio: Optional[str] = None       # optional field
    role: str = "user"              # default value

# This model defines:
#   name:  required, string, 1-100 characters
#   email: required, must be valid email format
#   age:   required, integer, between 1 and 149
#   bio:   optional, string or null
#   role:  optional, defaults to "user"
```

### Using Models in Endpoints

```python
@app.post("/users", status_code=201)
def create_user(user: UserCreate):    # <-- Pydantic does ALL validation
    # If we reach here, data is 100% valid
    # user.name, user.email, user.age are all correct types
    return {
        "message": "User created",
        "user": user.model_dump()     # convert to dict
    }

# POST /users
# Body: { "name": "Aashish", "email": "a@b.com", "age": 22 }
# -> 201 Created

# POST /users
# Body: { "name": "", "email": "not-an-email", "age": -5 }
# -> 422 Unprocessable Entity
# {
#   "detail": [
#     { "loc": ["body", "name"], "msg": "ensure this has at least 1 character" },
#     { "loc": ["body", "email"], "msg": "value is not a valid email address" },
#     { "loc": ["body", "age"], "msg": "ensure this value is greater than 0" }
#   ]
# }
# ALL validation errors returned at once! Not one at a time.
```

### Multiple Models for Different Operations

```python
# GOLDEN PATTERN: Separate models for Create, Update, and Response

# What the client sends to CREATE a user:
class UserCreate(BaseModel):
    name: str
    email: EmailStr
    password: str = Field(min_length=8)

# What the client sends to UPDATE a user (all fields optional):
class UserUpdate(BaseModel):
    name: Optional[str] = None
    email: Optional[EmailStr] = None
    bio: Optional[str] = None

# What the server sends BACK (never expose password!):
class UserResponse(BaseModel):
    id: int
    name: str
    email: EmailStr
    bio: Optional[str] = None
    created_at: datetime

    class Config:
        from_attributes = True   # allows ORM objects -> Pydantic model


# Usage:
@app.post("/users", response_model=UserResponse, status_code=201)
def create_user(user: UserCreate):
    # user.password is available here for hashing
    # but UserResponse does NOT include password
    # so it's automatically excluded from the response!
    ...

@app.patch("/users/{user_id}", response_model=UserResponse)
def update_user(user_id: int, user: UserUpdate):
    # Only non-None fields are updated (PATCH behavior)
    update_data = user.model_dump(exclude_unset=True)
    # update_data only contains fields the client actually sent
    ...
```

## Error Handling

```python
from fastapi import FastAPI, HTTPException, status, Request
from fastapi.responses import JSONResponse

app = FastAPI()

# Method 1: HTTPException (most common)
@app.get("/users/{user_id}")
def get_user(user_id: int):
    user = find_user(user_id)
    if not user:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="User not found"       # this becomes the response body
        )
    return user

# GET /users/999
# -> 404 { "detail": "User not found" }


# Method 2: Custom exception classes (production pattern)
class AppException(Exception):
    def __init__(self, status_code: int, code: str, message: str):
        self.status_code = status_code
        self.code = code
        self.message = message

class NotFoundException(AppException):
    def __init__(self, resource: str, resource_id: str):
        super().__init__(404, "NOT_FOUND", f"{resource} with id '{resource_id}' not found")

class ConflictException(AppException):
    def __init__(self, message: str):
        super().__init__(409, "CONFLICT", message)

# Register global exception handler:
@app.exception_handler(AppException)
async def app_exception_handler(request: Request, exc: AppException):
    return JSONResponse(
        status_code=exc.status_code,
        content={
            "error": {
                "code": exc.code,
                "message": exc.message,
            }
        },
    )

# Usage:
@app.get("/users/{user_id}")
def get_user(user_id: int):
    user = find_user(user_id)
    if not user:
        raise NotFoundException("User", str(user_id))
    return user

# -> 404 { "error": { "code": "NOT_FOUND", "message": "User with id '999' not found" } }
```

## Dependency Injection

```
Dependency Injection (DI) is one of FastAPI's MOST POWERFUL features.

What is it?
  -> A way to declare "this endpoint NEEDS this thing to work"
  -> FastAPI automatically provides (injects) that thing
  -> The "thing" can be: a DB session, auth user, config, etc.

Why it matters:
  -> No global variables floating around
  -> Each request gets its own isolated dependencies
  -> Easy to test (swap real DB for mock DB)
  -> Clean, explicit, composable
```

```python
from fastapi import Depends, FastAPI, HTTPException

app = FastAPI()

# Example 1: Database session dependency
def get_db():
    db = SessionLocal()    # create a DB session
    try:
        yield db           # provide it to the endpoint
    finally:
        db.close()         # close it after the request is done

@app.get("/users")
def get_users(db: Session = Depends(get_db)):    # <-- injected!
    return db.query(User).all()

# FastAPI:
#   1. Calls get_db()
#   2. Gets the db session
#   3. Passes it to get_users()
#   4. After response is sent, runs the finally block (closes session)
#   5. Each request gets its OWN session. No shared state.


# Example 2: Authentication dependency
def get_current_user(token: str = Header()):
    user = decode_token(token)
    if not user:
        raise HTTPException(status_code=401, detail="Invalid token")
    return user

@app.get("/profile")
def get_profile(user: User = Depends(get_current_user)):
    # If token is invalid -> 401 before this code runs
    # If token is valid -> user is available here
    return {"name": user.name, "email": user.email}


# Example 3: Pagination dependency (reusable)
def pagination_params(page: int = 1, limit: int = 20):
    return {"skip": (page - 1) * limit, "limit": limit}

@app.get("/users")
def get_users(pagination: dict = Depends(pagination_params)):
    # pagination = {"skip": 0, "limit": 20}
    ...

@app.get("/products")
def get_products(pagination: dict = Depends(pagination_params)):
    # Same pagination logic reused! No code duplication.
    ...
```

## Middleware

```python
from fastapi import FastAPI, Request
import time

app = FastAPI()

# Middleware runs on EVERY request, before and after the endpoint.

@app.middleware("http")
async def log_requests(request: Request, call_next):
    start_time = time.time()

    # BEFORE the endpoint runs:
    print(f"Incoming: {request.method} {request.url}")

    response = await call_next(request)   # <-- run the actual endpoint

    # AFTER the endpoint runs:
    duration = time.time() - start_time
    print(f"Completed: {response.status_code} in {duration:.3f}s")
    response.headers["X-Process-Time"] = str(duration)

    return response


# Request ID middleware (tracing):
import uuid

@app.middleware("http")
async def add_request_id(request: Request, call_next):
    request_id = str(uuid.uuid4())
    request.state.request_id = request_id

    response = await call_next(request)
    response.headers["X-Request-ID"] = request_id
    return response
```

## Authentication and Security

```python
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from jose import JWTError, jwt
from passlib.context import CryptContext
from datetime import datetime, timedelta

# Setup
SECRET_KEY = "your-secret-key-keep-this-safe"
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

pwd_context = CryptContext(schemes=["bcrypt"])
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="login")


# Password hashing:
def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)


# JWT token creation:
def create_access_token(data: dict) -> str:
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)


# Get current user from token (dependency):
def get_current_user(token: str = Depends(oauth2_scheme)):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        user_id = payload.get("sub")
        if user_id is None:
            raise HTTPException(status_code=401, detail="Invalid token")
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")

    user = get_user_from_db(user_id)
    if not user:
        raise HTTPException(status_code=401, detail="User not found")
    return user


# Login endpoint:
@app.post("/login")
def login(form_data: OAuth2PasswordRequestForm = Depends()):
    user = authenticate_user(form_data.username, form_data.password)
    if not user:
        raise HTTPException(status_code=401, detail="Invalid credentials")
    
    token = create_access_token(data={"sub": str(user.id)})
    return {"access_token": token, "token_type": "bearer"}


# Protected endpoint:
@app.get("/profile")
def get_profile(current_user: User = Depends(get_current_user)):
    return current_user

# The Swagger UI at /docs will show a "lock" icon on protected endpoints
# and provide a login form to test authentication!
```

```python
# crud.py - Database operations (Repository pattern)
from sqlalchemy.orm import Session
from models import User, Post
from schemas import UserCreate, PostCreate
from auth import hash_password

def create_user(db: Session, user: UserCreate) -> User:
    db_user = User(
        name=user.name,
        email=user.email,
        hashed_password=hash_password(user.password),
    )
    db.add(db_user)
    db.commit()
    db.refresh(db_user)
    return db_user

def get_user(db: Session, user_id: int) -> User | None:
    return db.query(User).filter(User.id == user_id).first()

def get_users(db: Session, skip: int = 0, limit: int = 20) -> list[User]:
    return db.query(User).offset(skip).limit(limit).all()

def create_post(db: Session, post: PostCreate, author_id: int) -> Post:
    db_post = Post(**post.model_dump(), author_id=author_id)
    db.add(db_post)
    db.commit()
    db.refresh(db_post)
    return db_post
```

## Background Tasks

```python
from fastapi import BackgroundTasks

# Background tasks run AFTER the response is sent.
# Perfect for: sending emails, logging, notifications.

def send_welcome_email(email: str, name: str):
    # This runs in the background, doesn't block the response

@app.post("/users", status_code=201)
def create_user(user: UserCreate, bg_tasks: BackgroundTasks):
    new_user = save_user(user)

    # Schedule email to be sent AFTER response:
    bg_tasks.add_task(send_welcome_email, user.email, user.name)

    # Response is sent immediately.
    # Email is sent in the background.
    # User doesn't wait for the email to finish.
```

```python
# routes/users.py - Full CRUD with all patterns applied

from fastapi import APIRouter, Depends, HTTPException, status, Query
from sqlalchemy.orm import Session
from database import get_db
from schemas.user import UserCreate, UserUpdate, UserResponse
from services.user_service import UserService

router = APIRouter(prefix="/api/v1/users", tags=["Users"])


@router.get("/", response_model=list[UserResponse])
def get_users(
    role: str = None,
    sort_by: str = Query(default="created_at"),
    page: int = Query(default=1, ge=1),
    limit: int = Query(default=20, ge=1, le=100),
    db: Session = Depends(get_db),
):
    service = UserService(db)
    return service.get_all(role=role, sort_by=sort_by, page=page, limit=limit)


@router.get("/{user_id}", response_model=UserResponse)
def get_user(user_id: int, db: Session = Depends(get_db)):
    service = UserService(db)
    user = service.get_by_id(user_id)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user


@router.post("/", response_model=UserResponse, status_code=201)
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    service = UserService(db)
    existing = service.get_by_email(user.email)
    if existing:
        raise HTTPException(status_code=409, detail="Email already registered")
    return service.create(user)


@router.patch("/{user_id}", response_model=UserResponse)
def update_user(user_id: int, user: UserUpdate, db: Session = Depends(get_db)):
    service = UserService(db)
    updated = service.update(user_id, user)
    if not updated:
        raise HTTPException(status_code=404, detail="User not found")
    return updated


@router.delete("/{user_id}", status_code=204)
def delete_user(user_id: int, db: Session = Depends(get_db)):
    service = UserService(db)
    deleted = service.delete(user_id)
    if not deleted:
        raise HTTPException(status_code=404, detail="User not found")
    return None
```

## Async vs Sync in FastAPI

```python
# FastAPI supports BOTH sync and async endpoints.

# SYNC (regular functions):
@app.get("/sync")
def sync_endpoint():
    # FastAPI runs this in a thread pool
    # Use for CPU-bound work or sync libraries
    result = some_sync_db_call()
    return result

# ASYNC (coroutines):
@app.get("/async")
async def async_endpoint():
    # FastAPI runs this in the event loop
    # Use for I/O-bound work (DB, HTTP calls, file I/O)
    result = await some_async_db_call()
    return result

# Rule of thumb:
#   -> If your DB driver is async (asyncpg, databases) -> use async def
#   -> If your DB driver is sync (psycopg2, SQLAlchemy sync) -> use def
#   -> Mixing async def with sync DB calls BLOCKS the event loop!
#   -> When in doubt, use regular def (FastAPI handles it safely)
```
